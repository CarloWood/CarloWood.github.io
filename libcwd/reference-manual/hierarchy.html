Class Hierarchy
This inheritance list is sorted roughly, but not completely, alphabetically:
 Clibcwd::alloc_ctAn object of type alloc_ct contains information about one allocated memory block
 Clibcwd::alloc_filter_ctAn allocated-memory filter class.An object of this type can be passed to list_allocations_on containing formatting information for the Overview Of Allocated Memory . It can also be passed as argument to the constructor of a marker_ct object in order to specify which allocations should not be capatured by the marker (and optionally made invisible)
 Clibcwd::buf2strPrint a (char) buffer with a given size to a debug ostream, escaping non-printable characters
 Clibcwd::channel_ctThis object represents a debug channel, it has a fixed label. A debug channel can be viewed upon as a single bit: on or off
 Clibcwd::char2strPrint a char to a debug ostream, escaping non-printable characters as needed
 Clibcwd::cwprintPrint an object to a debug stream without needing an operator<<
 Clibcwd::cwprint_usingPrint an object to an ostream using an arbitrary method of that object
 Clibcwd::debug_ctThe Debug Object class, this object represents one output device (ostream)
 Clibcwd::debug_string_ctA string class used for the debug output margin and marker.This type is used for the public attributes debug_ct::margin and debug_ct::marker of the debug object class
 Clibcwd::location_ctA source file location
 Clibcwd::marker_ctA memory allocation marker
 Clibcwd::object_file_ctAn object representing the main executable or a shared library
 Clibcwd::rcfile_ctThis object represents a runtime configuration file
 Clibcwd::type_info_ctClass that holds type information for debugging purposes.  Returned by type_info_of()
